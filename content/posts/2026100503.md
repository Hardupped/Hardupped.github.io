千聚完全兼容 OpenAI 接口格式。在 Java 里，你不需要装任何额外的SDK，直接用原生的 HttpURLConnection 或者流行的 okhttp3 库都行。

这里我展示最优雅的写法——用 Java 17+ 的 HttpClient 和 Jackson 库，实现真正的低代码接入：

java
import java.net.http.*;
import java.net.URI;
import com.fasterxml.jackson.databind.*;

public class GPT5CodexJavaDemo {
    public static void main(String[] args) throws Exception {
        HttpClient client = HttpClient.newHttpClient();
        
        // 关键点：只改base_url
        String url = "https://www.qianjuai.com/v1/chat/completions";
        
        // 构造请求体：极简的JSON结构
        String requestBody = """
        {
            "model": "gpt-5-codex",
            "messages": [
                {"role": "user", "content": "写一个Java程序，用Stream API过滤出List中所有奇数并求和"}
            ],
            "max_tokens": 1024,
            "temperature": 0
        }
        """;
        
        HttpRequest request = HttpRequest.newBuilder()
                .uri(URI.create(url))
                .header("Content-Type", "application/json")
                .header("Authorization", "Bearer your-api-key-here")
                .POST(HttpRequest.BodyPublishers.ofString(requestBody))
                .build();
        
        HttpResponse<String> response = client.send(request, HttpResponse.BodyHandlers.ofString());
        
        ObjectMapper mapper = new ObjectMapper();
        JsonNode root = mapper.readTree(response.body());
        String answer = root.get("choices").get(0).get("message").get("content").asText();
        
        System.out.println("GPT-5 Codex 生成的代码：\n" + answer);
    }
}


**整个代码只有不到 30 行**，没有复杂的依赖管理，没有冗余的配置，核心逻辑就是：构建请求 -> 发送请求 -> 解析响应。你甚至不需要懂大模型的工作原理。

这就是低代码接入的真正含义——不是让你去写一套复杂的接入框架，而是模型替你写完了最难的部分。
