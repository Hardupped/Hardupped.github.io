## 常见坑和避坑指南（企业关键）

在接入过程中，有几个容易踩的坑你必须留意：

**1. 如果返回 403 或认证错误**
这个最常见。检查 `apiKey` 是否复制完整（去掉多余空格），并且在[千聚api聚合站](https://www.qianjuai.com/)后台确认 API Key 处于“启用”状态。

**2. 流式输出卡顿 / 断连**
建议在代码中增加重试和超时设置：

javascript
const client = new OpenAI({
  baseURL: 'https://www.qianjuai.com/v1',
  apiKey: 'sk-...',
  timeout: 30000,
  maxRetries: 3,
});


**3. 模型名称写错**
[千聚api聚合站](https://www.qianjuai.com/)要求**完整准确**的模型 ID（比如 `claude-3-haiku-20240307`），简称或大小写错误都会导致模型不存在。你可以去后台“模型列表”页面确认当前可用的名称。

**4. 企业中如何安全分配 Key**
建议为每个项目或每个应用单独创建 API Key，通过千聚后台的“Key 管理”功能进行权限隔离，这样单 Key 出问题不影响其他业务。

**5. 非流式请求的高并发优化**
如果你是做批量数据分析，不走流式输出，可以调整 `maxRetries` 和并发数（比如用 `p-limit` 库控制并发量），避免短时间内超过分组限频。
