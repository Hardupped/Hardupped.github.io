## 这10个坑，我替你一个一个踩了

### 坑1：base_url写错了

很多人直接抄官方代码，`base_url` 攻写成了 `https://api.openai.com/v1`，结果提示网络连接失败。

**正确做法：** 换成千聚的 `https://www.qianjuai.com/v1`，国内直连不用代理。

### 坑2：proxy导致超时

如果你的机器配了全局代理，OpenAI官方请求会被重定向。千聚已经走国内直连，配置了代理反而冲突，导致`connect timeout`。

**正确做法：** 检查环境变量，去掉`http_proxy`和`https_proxy`，或者显式设置为空。

python

import os

os.environ.pop("http_proxy", None)

os.environ.pop("https_proxy", None)

### 坑3：api_key格式错误

有人从千聚后台复制Key时多了个空格，或者复制了带中文引号的代码，导致`AuthenticationError: Incorrect API key provided`。

**正确做法：** 检查Key前后有无空白符，单引号和双引号保持一致。特别建议在代码里直接打印Key的前几位和后几位来验证。

python

key = "sk-xxxxxx"

print(f"Key长度: {len(key)}，前5位: {key[:5]}")

### 坑4：流式输出与非流式输出混用

有些人的代码里同时用`stream=True`和`response.choices[0].message.content`，结果报了`AttributeError: 'ChatCompletionChunk' object has no attribute 'choices'`。

**正确做法：** `stream=True`只接受增量处理，`stream=False`才能用完整response。二选一别混搭。

### 坑5：messages格式错误

有人给用户输入直接传字符串，没套进字典里，报了`TypeError`。

python

# 错误写法

messages = [{"role": "user", "content": user_input}]  # ✅

messages = [user_input]  # ❌

### 坑6：系统提示过长

GPT-5pro对系统提示有token限制，但有些企业测试时把整本用户手册塞进去，结果被截断或报错。

**正确做法：** 精简系统提示，控制在2000token以内。如果确实需要长上下文，分段传给用户消息。

### 坑7：模型名拼错

有人写了`gpt5pro`少个连字符，`gpt-5`不带pro，`GPT-5Pro`大小写不一致，都会报`Model not found`。

**正确做法：** 去千聚后台看支持的模型列表，复制`gpt-5pro`这个完整名称。

### 坑8：使用已废弃的参数

GPT-5pro SDK有版本兼容性，有人抄了旧代码里的`engine`参数（已被`model`取代），导致`Unexpected keyword argument 'engine'`。

**正确做法：** 统一用`model="gpt-5pro"`，不用其他历史参数。

### 坑9：网络重试机制缺失

国内网络偶尔抖动很正常，但代码没有重试逻辑，一次超时就整个请求失败。

**正确做法：** 加个带指数退避的重试机制。

python

import time, random

def request_with_retry(client, max_retries=3):

    for i in range(max_retries):

        try:

            return client.chat.completions.create(...)

        except Exception as e:

            if i == max_retries - 1: raise e

            time.sleep(2 ** i + random.uniform(0, 1))

### 坑10：余额不足检查遗漏

钱花完了也不知道，报`Insufficient Quota`一脸懵。

**正确做法：** 调用前先查余额。千聚后台有清晰显示，代码里也可以加个预检查。

def ask_gpt5pro(system_prompt, user_input, stream=False):

    try:

        response = client.chat.completions.create(

            model="gpt-5pro",

            messages=[

                {"role": "system", "content": system_prompt},

                {"role": "user", "content": user_input}

            ],

            stream=stream,

            max_tokens=4096

        )

        if stream:

            for chunk in response:

                if chunk.choices[0].delta.content:

                    yield chunk.choices[0].delta.content

        else:

            return response.choices[0].message.content

    except Exception as e:

        print(f"请求失败: {e}")

        return None

# 测试使用
