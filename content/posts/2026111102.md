## 实战：Qwen3-Coder的四种典型调用模式

我整理了四种最常见的调用场景，每种都给出可直接复制的完整代码。记住，关键就是`base_url`和`api_key`这两行，其他地方和官方调用完全一致。

### 场景一：基础代码生成

这是最简单的用法，适合生成一段独立的代码：

python
response = client.chat.completions.create(
    model="qwen3-coder",  # 注意以千聚官方列表为准
    messages=[
        {"role": "system", "content": "你是一个专业的Python开发者，请生成高质量的代码。"},
        {"role": "user", "content": "写一个Python函数，用于计算斐波那契数列的第n项，要求使用动态规划。"}
    ],
    temperature=0.3,
    max_tokens=2000
)

print(response.choices[0].message.content)


### 场景二：多轮对话式代码编辑

适合在已有代码基础上进行修改、优化或解释：

python
messages = [
    {"role": "system", "content": "你是一位资深代码审查专家。"},
    {"role": "user", "content": "下面这段代码效率低下，请优化它：\n\npython\ndef sum_list(lst):\n    total = 0\n    for i in range(len(lst)):\n        total += lst[i]\n    return total\n"},
]

response = client.chat.completions.create(
    model="qwen3-coder",
    messages=messages,
    temperature=0.2,
    max_tokens=1500
)

print(response.choices[0].message.content)


### 场景三：流式输出

适合需要实时展示生成过程的场景，比如IDE代码补全或聊天窗口：

python
stream = client.chat.completions.create(
    model="qwen3-coder",
    messages=[
        {"role": "system", "content": "你是一个代码助手。"},
        {"role": "user", "content": "请用Python写一个装饰器，用于计算函数执行时间。"}
    ],
    stream=True,
    max_tokens=2000
)

for chunk in stream:
    if chunk.choices[0].delta.content is not None:
        print(chunk.choices[0].delta.content, end="")


### 场景四：批量代码处理

适合在后台批量处理多个代码文件：

python
import asyncio
from openai import AsyncOpenAI

aclient = AsyncOpenAI(
    api_key="sk-你的千聚API密钥",
    base_url="https://www.qianjuai.com/v1"
)

async def analyze_code(code_snippet):
    response = await aclient.chat.completions.create(
        model="qwen3-coder",
        messages=[
            {"role": "system", "content": "分析以下代码，找出潜在bug和安全漏洞。"},
            {"role": "user", "content": f"python\n{code_snippet}\n"}
        ],
        max_tokens=1000
    )
    return response.choices[0].message.content

# 批量处理
codes = ["print('hello')", "import os; os.system('rm -rf /')"]
tasks = [analyze_code(code) for code in codes]
results = asyncio.gather(*tasks)
print(results)

## 最常遇到的报错及解决方案

尽管接入流程已经简化到极致，但新手仍然可能遇到几个常见错误。我在这里一并解决，真正做到“保姆级”。

| 报错信息 | 问题原因 | 解决方法 |
|-----------|----------|----------|
| `ConnectionError` | 网络无法连接千聚API | 检查网络是否正常，能否访问`www.qianjuai.com` |
| `AuthenticationError` | API密钥错误或过期 | 重新在千聚后台生成密钥并替换 |
| `NotFoundError` | 模型名称写错 | 去千聚模型列表确认正确的模型ID |
| `RateLimitError` | 请求频率过高 | 在请求间添加短暂延时，或升级账户 |
| `BadRequestError` | 消息格式或参数错误 | 检查`messages`结构是否符合规范 |

其中，`ConnectionError`是最常见的，也是最能凸显千聚优势的地方——换成官方API，同样的代码在网络受限环境下就会报这个错；换成千聚，国内直连即可。

## 进阶：参数调优与成本控制

接入成功只是第一步，要想在实际项目中稳定高效地使用Qwen3-Coder，参数调优是关键。

**temperature参数**：代码生成类任务建议设为0.1-0.3，保持高度确定性；创意性代码或代码解释类任务可以设为0.5-0.7。

**max_tokens参数**：控制输出长度。Qwen3-Coder支持长上下文，但没必要每次都设置过大的值。生成单函数设为500-1000，生成完整模块设为2000-4000。

**streaming模式**：建议在交互式场景（如IDE插件）中必须开启，用户体验完全不一样；在批量处理场景中关闭，节省token。

关于成本，通过千聚调用Qwen3-Coder的价格优势非常明显。以Qwen3-Coder-32B为例，它的推理成本只有GPT-4系列的几分之一，但代码生成质量接近。你在千聚后台可以实时查看每次调用的token消耗和费用，完全透明。
