### 第二步：用千聚API搭建问答后端（含代码示例）

系统搭建的核心是把千聚的接口接上你的业务逻辑。这里我提供一个最简的Python代码示例，用openai官方库直接调用千聚接口，实现一个基础的问答函数：

import os
from openai import OpenAI

# 关键配置：只需修改base_url和api_key
client = OpenAI(
    api_key="你的千聚密钥",  # 替换成你复制的密钥
    base_url="https://www.qianjuai.com/v1"  # 接口地址

def smart_query(user_input, model_name="gpt-4o-mini"):
    """
    通用问答函数，可切换不同模型
    """
    response = client.chat.completions.create(
        model=model_name,  # 这里可以随时改成 "deepseek-r1" 或 "claude-3.5-sonnet"
        messages=[
            {"role": "system", "content": "你是一个专业的客服助手，请用简洁准确的中文回答用户提问。"},
            {"role": "user", "content": user_input}
        ],
        temperature=0.3,  # 客服场景建议低温度，保持回答准确性
        max_tokens=500,
        stream=False  # 如果需要流式输出，改成True
    )
    return response.choices[0].message.content


**核心逻辑解析：**
- **base_url**：指向千聚的兼容接口，这是“安全不封号”的关键——你的请求通过国内服务器转发，不直接触碰OpenAI官方API，极大降低了因IP或风控问题被封的风险。
- **model参数**：你想用哪个模型，直接把名字写进去就行。千聚的模型列表里支持什么，这里就能写什么。
- **system prompt**：这是问答系统的“灵魂”。你可以在这里定义回复语气、知识边界、甚至回答长度。

把这个函数封装成一个API接口，你的前端网页、微信小程序、或钉钉机器人就能直接调用了。
