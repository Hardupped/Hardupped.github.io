## 玩转多模型并行调用：这才是中转站的王炸功能

好，现在你已经能调用一个模型了。但这篇文章的标题是“玩转多模型并行调用”，到底什么是并行调用？为什么说它是王炸？

简单说，**并行调用就是让多个模型同时为你处理一个任务，然后你把它们的结果放在一起对比，从中选出最好的那个。**

**这样做的好处是什么？**

1.  **效率飙升**：你想知道“写一部科幻小说开头”，GPT-4o 擅长创意，Claude 擅长逻辑，DeepSeek 文风有特色。把它们同时叫出来，1分钟后，你手里就有3个完全不同风格的开头，直接选最好的那个，不用一个一个等。
2.  **成本最优**：有些简单任务（比如分类邮件），用 GPT-4o 太贵，用 DeepSeek-V3 又快又便宜。你可以通过代码判断，简单任务走便宜模型，复杂任务走昂贵模型，最大化利用你充的钱。
3.  **结果最优**：做数据分析时，让 Gemini 和 GPT-4o 同时分析同一份数据，对比它们的结论差异，可以帮你发现单个模型可能忽略的盲点。

**代码怎么实现？其实很简单！**

你只需要在你的代码里，创建两个不同的 `client` 对象或者在一个循环里切换 `base_url` 和 `model` 的参数，然后用多线程或异步的方式同时发送请求。

用Python的多线程举例（伪代码，但逻辑一目了然）：

python
import concurrent.futures

def call_ai(model_name):
    client = OpenAI(api_key="你的key", base_url="https://www.qianjuai.com/v1")
    response = client.chat.completions.create(
        model=model_name,
        messages=[{"role": "user", "content": "用简洁的语言解释什么是黑洞"}],
    )
    return response.choices[0].message.content

with concurrent.futures.ThreadPoolExecutor() as executor:
    # 同时调用三个不同的模型
    results = executor.map(call_ai, ['gpt-4o', 'deepseek-chat', 'claude-3-5-sonnet-20241022'])

for result in results:
    print("结果: ", result)
