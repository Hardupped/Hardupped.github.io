## 发送你的第一个请求

好，现在开始写代码。目标是调用DeepSeek的 `deepseek-chat` 模型，问一句“你好，你是谁？”

### 核心代码示例

java
import com.theokanning.openai.OpenAiService;
import com.theokanning.openai.completion.chat.ChatCompletionRequest;
import com.theokanning.openai.completion.chat.ChatMessage;
import com.theokanning.openai.completion.chat.ChatMessageRole;

import java.util.ArrayList;
import java.util.List;

public class DeepSeekTest {
    public static void main(String[] args) {
        // 1. 配置base_url和API Key
        String baseUrl = "https://www.qianjuai.com/v1";
        String apiKey = System.getenv("DEEPSEEK_API_KEY");
        
        if (apiKey == null || apiKey.isEmpty()) {
            System.out.println("错误：未找到环境变量 DEEPSEEK_API_KEY，请在环境变量中配置。");
            return;
        }

        // 2. 创建OpenAiService实例
        OpenAiService service = new OpenAiService(apiKey, Duration.ofSeconds(60));
        
        // 3. 构建请求
        List<ChatMessage> messages = new ArrayList<>();
        messages.add(new ChatMessage(ChatMessageRole.USER.value(), "你好，你是谁？"));
        
        ChatCompletionRequest request = ChatCompletionRequest.builder()
                .model("deepseek-chat")
                .messages(messages)
                .temperature(0.7)
                .maxTokens(500)
                .build();
        
        // 4. 发送请求并获取响应
        try {
            ChatMessage responseMessage = service.createChatCompletion(request)
                    .getChoices().get(0).getMessage();
            System.out.println("DeepSeek: " + responseMessage.getContent());
        } catch (Exception e) {
            System.out.println("调用失败，错误信息：" + e.getMessage());
            e.printStackTrace();
        }
    }
}


### 关键点解释

- `baseUrl`: 一定要设置成千聚的地址。不设置的话，默认会连接到OpenAI官网，在国内连不上，会报超时。
- `model`: 填 `deepseek-chat` 就行。千聚的DeepSeek是满血版，不是阉割版。
- `temperature` 和 `maxTokens`: 这两个参数可以按需调整，不是必须的，但不传可能用默认值。如果觉得回答太随机，可以降低temperature。
- `Duration.ofSeconds(60)`: 设置超时时间。第一次请求可能慢一点，60秒足够。

## 常见错误排查指南

### 错误1：400 Bad Request
- **原因**：请求参数格式错误，比如model名称写的不对。
- **解决方法**：检查 `model` 字段是否填了 `deepseek-chat`，或者其他受支持的模型名。千聚API文档里有完整的模型列表，再去确认一下。

### 错误2：401 Unauthorized
- **原因**：API Key没有传，或者传错了。
- **解决方法**：检查代码中 `apiKey` 的值是否正确。检查环境变量是否正确读取。

### 错误3：502 Bad Gateway
- **原因**：千聚API服务端正在维护或临时故障，极少数情况。
- **解决方法**：等几分钟重试。如果多次出现，可以去千聚官网看公告。

### 错误4：流式输出代码不工作
- **原因**：用的 `openai-java` 版本过低，不支持流式接口。
- **解决方法**：升级到0.18.2或更高版本。同时，流式请求需要用 `service.streamChatCompletion()` 方法，返回值不一样。

## 高级玩法：让代码稳定运行

### 1. 添加重试机制

网络不稳定的时候，加个重试会让程序健壮很多。

java
int maxRetries = 3;
int retryCount = 0;
boolean success = false;

while (retryCount < maxRetries && !success) {
    try {
        // 上面发送请求的代码
        success = true;
    } catch (Exception e) {
        retryCount++;
        System.out.println("第" + retryCount + "次调用失败，3秒后重试...");
        Thread.sleep(3000);
    }
}


### 2. 异步调用

如果需要并发请求，可以用线程池。

java

        // 执行上面的请求代码
    });
}
executor.shutdown();
