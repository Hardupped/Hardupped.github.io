### 第三步：写一个最简单的请求

现在我们来写一个最简化的Java示例，用千聚的API调用Kimi模型：

java
import com.azure.ai.openai.OpenAIClient;
import com.azure.ai.openai.OpenAIClientBuilder;
import com.azure.ai.openai.models.ChatCompletions;
import com.azure.ai.openai.models.ChatCompletionsOptions;
import com.azure.ai.openai.models.ChatRequestAssistantMessage;
import com.azure.ai.openai.models.ChatRequestUserMessage;
import com.azure.core.credential.KeyCredential;

import java.util.Arrays;

public class KimiDemo {
    public static void main(String[] args) {
        // 替换为你的API Key和base_url
        String apiKey = "你的千聚API Key";
        String endpoint = "https://www.qianjuai.com/v1";

        OpenAIClient client = new OpenAIClientBuilder()
                .credential(new KeyCredential(apiKey))
                .endpoint(endpoint)
                .buildClient();

        // 注意：Kimi模型在千聚平台上的模型名称可能为"kimi-*-*"，请根据后台提示填写
        String modelName = "kimi-latest";
        ChatCompletionsOptions options = new ChatCompletionsOptions(Arrays.asList(
                new ChatRequestUserMessage("你好，请介绍一下你自己。")
        ));
        options.setModel(modelName);

        ChatCompletions completions = client.getChatCompletions(modelName, options);
        
        System.out.println("Kimi回复: ");
        completions.getChoices().forEach(choice -> {
            System.out.println(choice.getMessage().getContent());
        });
    }
}


这段代码看起来很简单，对吧？其实大多数大模型API的接入都可以用这个模板，只是模型名和请求参数不同而已。

## 避坑指南：小白最容易踩的5个雷

写代码容易，跑起来才难。下面这几个坑我踩过，你大概率也会遇到。

### 坑1：Base URL写错了

很多文档里给的示例`base_url`是`https://api.openai.com`，你用千聚的api key去调，肯定报401 Unauthorized。**一定要改成 `https://www.qianjuai.com/v1`**，别忘了最后的`/v1`。

### 坑2：模型名写错了

Kimi模型在千聚平台上的模型名称不一定叫"kimi"，可能叫"kimi-*"，也可能会跟其他大模型在一起。**去后台的模型列表里找到确切的名字**，别瞎猜。

### 坑3：依赖版本冲突

如果你之前项目里用了旧版的OpenAI SDK，可能跟新版的冲突。建议用最新的稳定版，或者直接把项目清干净重新构建。

### 坑4：网络问题

虽然千聚是国内直连，但如果你的公司网络限制了某些端口或域名，也可能连不上。**先试试在浏览器里访问 `www.qianjuai.com`，看看能不能打开**。

### 坑5：API Key硬编码

新手一个常见错误就是把API Key直接写在代码里，然后提交到GitHub。**用环境变量或配置文件来存储**，别让Key泄漏出去。

## 调试技巧：怎么让自己不抓狂

跑通Demo只是第一步，接下来你肯定要调试各种参数。这几招能救你命。

### 技巧1：用Postman先测一遍

别一上来就写完整的Java代码。用Postman，填上 `Base URL = https://www.qianjuai.com/v1`，选POST，加上Authorization Headers，发一个最简单的请求。能收到回复说明网络和Key都没问题，再回去写代码。

### 技巧2：日志级别开到最大

在Java代码里把HTTP客户端的日志级别调到DEBUG或ALL，这样你能看到具体的请求和响应内容，报错信息一目了然。

xml
<!-- 在logback.xml或application.properties中 -->
<logger name="com.azure" level="DEBUG"/>


### 技巧3：学会看错误码

- 401：API Key错误或未授权
- 404：Base URL或模型名写错了
- 429：请求频率太高，加个重试机制
- 500：服务端问题，等几分钟重试

### 技巧4：使用环境变量

把API Key放到环境变量里，代码里用`System.getenv("KIMI_API_KEY")`去取，这样既安全又方便切换不同环境的Key。
