### 代码实现：完整Java示例

下面的代码展示了如何向千聚API发送一个Kimi对话请求，并以流式方式获取AI回复。我们使用标准的OpenAI `v1/chat/completions` 接口。

java
import okhttp3.*;
import org.json.JSONArray;
import org.json.JSONObject;

import java.io.IOException;
import java.util.concurrent.TimeUnit;

/**
 * 千聚 api 聚合平台 - Kimi API Java 调用示例
 * 本示例使用了OKHTTP3和org.json库，请确保Maven/Gradle中已添加相应依赖。
 */
public class KimiApiClient {

    // ---------- 配置你自己的参数 ----------
    private static final String API_KEY = "sk-你的真实APIKey"; // 替换为千聚生成的Key
    private static final String BASE_URL = "https://www.qianjuai.com/v1"; // 不可变
    // ------------------------------------

    // ---------- 核心方法：发送流式聊天请求 ----------
    public static String sendChatRequest(String userMessage) throws IOException {
        // 1. 构建请求体 JSON
        JSONObject requestBody = new JSONObject();
        requestBody.put("model", "kimi-deep-answer"); // 假设的Kimi模型标识，请以千聚官方列表为准

        JSONArray messages = new JSONArray();
        JSONObject systemMessage = new JSONObject();
        systemMessage.put("role", "system");
        systemMessage.put("content", "你是一个有用的AI助手。请用中文回答用户的问题。");

        JSONObject userObject = new JSONObject();
        userObject.put("role", "user");
        userObject.put("content", userMessage);

        messages.put(systemMessage);
        messages.put(userObject);
        requestBody.put("messages", messages);
        requestBody.put("stream", true); // 启用流式输出
        requestBody.put("max_tokens", 1024); // 控制回复长度

        // 2. 构建HTTP请求
        OkHttpClient client = new OkHttpClient.Builder()
                .connectTimeout(30, TimeUnit.SECONDS)
                .readTimeout(30, TimeUnit.SECONDS)
                .build();

        Request request = new Request.Builder()
                .url(BASE_URL + "/chat/completions")
                .addHeader("Authorization", "Bearer " + API_KEY)
                .addHeader("Content-Type", "application/json")
                .post(RequestBody.create(
                        MediaType.parse("application/json"),
                        requestBody.toString()
                ))
                .build();

        // 3. 发送请求并处理流式响应
        StringBuilder fullResponse = new StringBuilder();
        try (Response response = client.newCall(request).execute()) {
            if (!response.isSuccessful()) {
                System.err.println("请求失败，HTTP状态码: " + response.code());
                // 打印错误详细信息（生产环境中请记录日志）
                if (response.body() != null) {
                    System.err.println("错误详情: " + response.body().string());
                }
                return null;
            }

            // 读取流式响应（SSE格式）
            ResponseBody body = response.body();
            if (body != null) {
                okio.BufferedSource source = body.source();
                String line;
                while ((line = source.readUtf8Line()) != null) {
                    if (line.startsWith("data: ")) {
                        String data = line.substring(6);
                        if ("[DONE]".equals(data)) {
                            break; // 流式结束
                        }
                        try {
                            JSONObject jsonData = new JSONObject(data);
                            String deltaContent = jsonData
                                    .getJSONArray("choices")
                                    .getJSONObject(0)
                                    .optJSONObject("delta")
                                    .optString("content", "");
                            fullResponse.append(deltaContent);
                        } catch (Exception e) {
                            // 解析异常通常发生在特殊事件，忽略或记录
                        }
                    }
                }
            }
        }
        return fullResponse.toString();
    }

    // ---------- 主方法测试 ----------
    public static void main(String[] args) {
        try {
            String reply = sendChatRequest("你好！请用中文介绍下你自己有哪些能力？");
            System.out.println("Kimi回答: " + reply);
        } catch (IOException e) {
            e.printStackTrace();
            System.err.println("调用失败，请检查网络或连接配置。");
        }
    }
}

### 代码讲解与注意事项

1.  **替换API Key**：将`sk-你的真实APIKey`换成你在千聚生成的Key。**不要**在代码里硬编码生产环境的Key，建议从环境变量或配置文件读取。
2.  **模型名称必须准确**：上面用的`kimi-deep-answer`是假设名称。**使用前请登录千聚后台，在“模型列表”中确认Kimi的模型标识名**，再用实际名称替换。
3.  **流式解析**：代码中读取了`data:`开头的SSE事件流。每次`delta`对象的`content`字段就是Kimi返回的增量文字。当遇到`data: [DONE]`时表示回复结束。
4.  **错误处理**：如果HTTP状态码不是200，打印错误详情，问题通常出在Key无效、模型名不对或网络问题。
5.  **依赖准备**：本项目依赖`okhttp3`和`org.json`库。如果你使用Maven，在`pom.xml`里添加：

<dependency>
    <groupId>com.squareup.okhttp3</groupId>
