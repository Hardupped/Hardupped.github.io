### 1. 连接超时
直接调用 OpenAI API 时，由于国内网络对境外服务器的访问存在大量丢包和延迟，请求经常在 30 秒内无法建立连接，直接被操作系统或 SDK 判定为超时。

### 2. 认证失败
认证失败不一定是你填错了 Key。部分中间代理或反向代理服务修改了请求头中的认证字段，或者接口地址不是标准的 OpenAI 格式，导致请求被 OpenAI 服务器拒绝。

### 3. 速率限制
官方 API 对每个 Key 每分钟的请求次数（RPM）和 Token 消耗（TPM）有严格限制。当你用同一个 Key 同时跑多个客户端或多个流程时，很容易撞墙。
