### 第二步：修改 Base URL（最核心的步骤）

这是最最偷懒的一步。你只需要在构建 HTTP Client 的时候，把 `baseUrl`（或者叫 `apiBase`）换成千聚的地址就行了。其他的代码逻辑，包括构造请求、处理响应，**一个字都不用改**。

看代码：

java
import com.theokanning.openai.OpenAiService;
import com.theokanning.openai.completion.chat.ChatCompletionRequest;
import com.theokanning.openai.completion.chat.ChatMessage;
import com.theokanning.openai.completion.chat.ChatCompletionResult;
import java.util.List;

public class O3Example {
    public static void main(String[] args) {
        // 1. 你的 API Key（去千聚注册获取）
        String apiKey = "sk-你的千聚APIkey";

        // 2. 核心！把请求地址指向千聚
        //    OpenAI 官方的默认地址是 https://api.openai.com/v1
        //    我们改成：https://www.qianjuai.com/v1
        OpenAiService service = new OpenAiService(apiKey, Duration.ofSeconds(60));

        // 手动设置 baseUrl 的方法（部分版本需要）
        // service.setBaseUrl("https://www.qianjuai.com/v1");

        // 3. 构建 o3 的请求
        ChatCompletionRequest request = ChatCompletionRequest.builder()
                .model("o3-mini") // 或 "o3", "o1" 等
                .messages(List.of(
                        new ChatMessage("system", "你是资深Java技术专家"),
                        new ChatMessage("user", "用Java写一段Stream API的示例")
                ))
                .build();

        // 4. 发送请求，获取结果
        ChatCompletionResult result = service.createChatCompletion(request);

        // 5. 打印答案
        result.getChoices().forEach(choice -> {
            System.out.println(choice.getMessage().getContent());
        });
    }
}


看到没，就是这么简单。你把 `model` 参数改成 `"o3-mini"`，把 `baseUrl` 改成千聚的，你的 Java 项目就能用上最新的 O3 了。

### 第三步：处理流式响应

如果你想实现像 ChatGPT 那样一个字一个字往外蹦的效果，用流式 (Stream) 调用也很简单：

java
// 使用 OpenAiService 的 stream 方法
service.streamChatCompletion(request)
    .doOnNext(chunk -> {
        // chunk 里包含了每次返回的增量文本
        String delta = chunk.getChoices().get(0).getMessage().getContent();
        if (delta != null) {
            System.out.print(delta); // 逐字打印
        }
    })
    .blockingSubscribe(); // 等待流结束


用这种 `doOnNext` + `blockingSubscribe` 的方式，在 Spring Boot 项目里做流式输出非常优雅，既能实现实时响应，又不会卡住主线程。
