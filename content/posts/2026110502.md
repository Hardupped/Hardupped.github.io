很多人抱怨“一接中转就封号”，问题往往不出在中转站服务商本身，而是出在用户自己的“接入方式”上。我总结了三个最常见的作死行为：

### 1. 使用非官方API的中转，导致“身份混淆”

很多小平台提供的是“拼车”或“代购”模式。多个用户共用一个 Google 或 OpenAI 的排他性 API Key。这种情况下，如果某一个用户的并发、频率或者错误参数触发了提供商的审计规则，整把 Key 就会被直接毙掉。你明明什么都没做错，但号就那么没了。

### 2. 频率控制不当，触发限流机制

Gemini 2.5 Flash 虽然是高性价比模型，但对于 API 调用仍然有 QPS（每秒请求数）限制。接入中转站后，如果脚本没有设置好重试机制和退避策略，短时间内高频调用，轻易就会被上游识别为异常流量而被熔断。

### 3. 低质代理链的“共用节点”

一些低质的中转服务，会共用大量代理节点。一旦某个节点因为其他用户的滥用行为被 Google 拉黑，你在那个节点上的所有请求都会失败，甚至被牵连封禁。
