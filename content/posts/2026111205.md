### 第三步：写核心调用代码

下面这个完整的Java示例，从发起请求到接收流式输出，全都给你写清楚了：

java
import com.theokanning.openai.OpenAiService;
import com.theokanning.openai.completion.chat.ChatCompletionRequest;
import com.theokanning.openai.completion.chat.ChatMessage;
import com.theokanning.openai.completion.chat.ChatMessageRole;
import java.time.Duration;
import java.util.Arrays;

public class GrokJavaIntegrationDemo {
    
    private static final String BASE_URL = "https://www.qianjuai.com/v1";
    private static final String MODEL = "grok-1"; // 千聚已全系支持Grok模型
    
    public static void main(String[] args) {
        // 从环境变量获取API Key
        String apiKey = System.getenv("QIANJU_API_KEY");
        if (apiKey == null) {
            System.err.println("错误：请设置QIANJU_API_KEY环境变量");
            return;
        }
        
        // 创建OpenAI服务实例，指定千聚中转地址
        OpenAiService service = new OpenAiService(
            apiKey,
            Duration.ofSeconds(60)
        );
        
        // 构建聊天请求
        ChatCompletionRequest request = ChatCompletionRequest.builder()
            .model(MODEL)
            .messages(Arrays.asList(
                new ChatMessage(ChatMessageRole.SYSTEM.value(), "你是一个专业的Java技术顾问。"),
                new ChatMessage(ChatMessageRole.USER.value(), "请解释Java中的CompletableFuture用法。")
            ))
            .maxTokens(500)
            .temperature(0.7)
            .build();
        
        // 发送请求并获取流式输出（推荐用于长回复）
        try {
            service.streamChatCompletion(request)
                .blockingForEach(chunk -> {
                    if (chunk.getChoices() != null && !chunk.getChoices().isEmpty()) {
                        var choice = chunk.getChoices().get(0);
                        if (choice.getMessage() != null && choice.getMessage().getContent() != null) {
                            System.out.print(choice.getMessage().getContent());
                        }
                    }
                });
            System.out.println("\n\n✅ Grok接口接入成功！");
        } catch (Exception e) {
            System.err.println("❌ 请求失败：" + e.getMessage());
        }
    }
}

### 坑1：SSL证书错误
**现象**：`javax.net.ssl.SSLHandshakeException`

**原因**：使用了一些未配置完整证书链的代理或自签名证书服务器。

### 坑2：超时/连接失败
**现象**：`java.net.ConnectException: Connection timed out`

**原因**：网络环境限制导致无法直连，或者代码里设置的超时太短。

**解决方案**：
因为千聚支持国内直连，所以绝大多数情况是超时设置太短。我在上面代码里用的`Duration.ofSeconds(60)`就很稳妥。如果项目要求更低的延迟，可以调整到30秒，但不要低于15秒。

### 坑3：API Key格式错误
**现象**：HTTP 401 Unauthorized

**原因**：把千聚的API Key贴错了格式，或者包含了空格、换行符。

**解决方案**：
在千聚官网复制Key时，一定要用官方提供的复制按钮。最保险的做法是：

}


### 坑4：模型名称错误
**现象**：`HTTP 404 Model not found`

**原因**：在代码里写的模型名和千聚平台上实际支持的名称不一致。
